## 最新结论（2026-07-11）

当前分支已经从“准备 specialist + response KD”转成 **response-structure 负结果报告**。
Push Cube 只留下 signed-target-coordinate 的窄可行性信号；Reacher 的 magnitude
partition 被 oracle gate 拒绝；Robot Arm 使用官方 dense checkpoint 跑完 Gate A 后

现在可保留的产出是：`ActionSchema`、`ResponseProbe`、`CounterfactualEvaluator`、
随机分组对照、三种子聚合、response heatmap/distribution、ID/OOD degradation summary
和 Gate A 机器决策。正式失败报告见：

```bash
docs/negative_result_response_structure.md
docs/negative_result_summary_zh.md
```

除非未来引入真正高 DoF、异构动作的数据集并先通过 Gate A，否则本项目不再把
specialist/KD 作为活动主线。

## 1. Idea 现在的判断

这个项目目前最值得保留的核心问题不是“action latent 一定比 raw action 好”，而是：

> student 在被压缩或蒸馏之后，能不能继续保留 teacher 对不同动作的响应差异。

换句话说，项目真正要验证的是 **action response preservation**。这比单纯说“做 latent action”更有研究价值，因为 latent action、world model distillation、specialist-to-generalist distillation 都已经有相邻工作；更锋利的切口应该是：**在 ACWM-Phys 这种 action-conditioned world model benchmark 上，用 response KD 检查模型输出是否随 action 改变**。在没有 matched counterfactual ground truth 时，这不等于证明 controllability 或物理正确性。

当前判断：

- 值得做成一个小而清楚的研究项目。
- 不建议一开始做大模型或复杂 VQ/RVQ tokenizer。
- 第一阶段应该先用 continuous action encoder、small/large teachers、response KD 和 action ablation 跑通闭环。
- 如果 specialist teacher 在自己的 action bucket 内都不能超过 generalist，那么后续蒸馏就应该暂停或重设 split。

最重要的风险是：模型可能视觉指标不错，但其实几乎不响应 action。因此后续评估必须包含 `zero / reverse / scale_0_5` 这类 action ablation，而不能只看 MSE、PSNR、SSIM。

## 5. 成败标准

继续推进的信号：

- latent baseline 不弱于 raw-action baseline
- small / large specialist 在自己的 bucket 内超过 generalist
- response KD 比普通 KD 更能区分 `a`、`zero`、`reverse`、`0.5a`
- action ablation 的差异变大，但真实 action rollout 的视觉指标没有明显崩

应该暂停或重设方案的信号：

- specialist teacher 不如 generalist
- `pred(a)`、`pred(0)`、`pred(-a)` 仍然几乎一样
- response KD 只是制造更大的预测差异，但真实 rollout 更差
- action latent 只是在增加参数量，没有带来 controllability 改善

## 6. 当前结论

这个 idea 现在已经从“概念判断”推进到了“可实验工程骨架”。下一步不是继续堆模块，而是跑最小实验，回答一个非常具体的问题：

> response KD 是否真的比普通 prediction KD 更能保留 action-conditioned world model 的动作响应？

如果 Push Cube / Reacher 上能看到这个信号，再考虑更复杂的 split、VQ/RVQ action tokenizer、Robot Arm / Cloth Move 等高维或复杂动力学环境。

## 7. 2026-07-11 验证更新：方向 specialist response KD 有可复现的 work 信号

这次没有继续堆模块，而是先修正了验证本身的两个问题：

- action ablation 现在在 `a / zero / reverse / 0.5a` 间复用**同一份初始扩散噪声**。此前每个条件独立采样，模型即使忽略 action 也会因为噪声不同而看起来“有响应”。
- 数据集现在会跳过缺失视频，并在真正读取不到视频时失败，而不是悄悄补全零帧。发现 OOD Push Cube 本地缺 34/50 个视频后，已把所需文件下载到独立数据盘 `/root/autodl-tmp/acwm-phys/`，再以符号链接接入；系统盘没有因此增长。训练 split 也从 584/1500 补到 1129/1500 个可用视频，剩余 371 个因 HF 未认证请求反复 TLS 断流未完成；扩大训练前必须继续补齐并重估 bucket。

### 7.1 先验证 latent interface

在相同的 100 step、256 trajectories、DiT-S 控制实验中：

| 模型 | ID true-action MSE | ID PSNR |
| --- | ---: | ---: |
| raw action | 0.01742 | 17.59 |
| continuous MLP latent action | 0.01594 | 17.97 |

因此 continuous action latent 作为 interface 有小但明确的正向视觉信号（MSE 约降低 8.5%）；这不是纯粹增加参数后崩坏的结果。

### 7.2 原始 ensemble 为什么不 work

Push Cube 的每步 action L2 magnitude 恒为 1，所以所谓 `small / large magnitude` split 实际退化成 absolute target action 第一维的 signed-coordinate split；它不是已经证明的 motion direction split。现有 `small` teacher 有明显 response，但 `large` teacher 在配对噪声 ablation 下几乎 action-blind。把这两个 teacher 都用于 KD，或仅把 `mu_resp` 提高到 2.0，都只能得到约 `5.6e-6` 的 `MSE(pred(a), pred(-a))`，没有可靠的 action-response 改善。

结论：**当前“所有 bucket ensemble + velocity response KD”版本不 work**，根因是未通过 oracle gate 的 teacher 被当作监督信号，而不是 response 权重太小。

### 7.3 可工作的改进版

保留唯一已通过 feasibility oracle check 的 signed-target-coordinate specialist，并且只在该 bucket 内训练 response-KD student。当前不完整训练 split 下的临时阈值是 `signed_action_0 = -0.018154`，不能宣称为稳定阈值。评估固定输入、固定初始噪声、10 denoising steps、seed 123、8 batches。Push Cube 中 generic `reverse` 实际表示 **negated-target action**，不是已证明的反向运动：

| split / signed-target-coordinate bucket | 模型 | true-action MSE | absolute paired response `MSE(pred(a), pred(-a))` |
| --- | --- | ---: | ---: |
| ID / small | latent-only baseline | 0.016894 | 1.975e-7 |
| ID / small | single-specialist response KD | 0.016965 | 2.690e-5 |
| OOD / small | latent-only baseline | 0.013030 | 2.068e-7 |
| OOD / small | single-specialist response KD | 0.012920 | 3.159e-5 |

因此，改进版的 `reverse` action response 在 ID 提升约 **136×**、OOD 提升约 **153×**；视觉 rollout 没有明显牺牲，OOD 反而略好。这是当前 idea 的 work 信号，但它只支持以下更窄的表述：

> 在 action-responsive 的 signed-target-coordinate specialist regime 内，response KD 可以使 student 的 paired counterfactual action response 增强，同时 true-action rollout quality 基本不变。

ID absolute response 从 `1.975e-7` 增至 `2.690e-5`；OOD 从 `2.068e-7` 增至 `3.159e-5`。现有持久化结果没有保存 teacher absolute response，因此 teacher response 和 student/teacher retention 暂记为 unavailable，不能倒推。

### 7.4 当前局限与下一步

- 目前只有 Push Cube、一个 signed-target-coordinate bucket、一个随机种子和 100 step feasibility run，不能声称通用方法。
- 当前本机 `ind_train` 仍未完全补齐（1129/1500 可用）。ID/OOD eval split 已可用，但任何新训练、teacher 对比或正式表格都要先补齐 train split 并重算 bucket。
- `-a` 与 `0.5a` 没有对应的真实 counterfactual GT，因此现在证明的是 response preservation，不是反事实物理正确性。
- student 的 response 仍小于 teacher；需要多 seed 和更长训练检验是否会继续缩小差距。
- Push Cube 不适合 magnitude specialist。下一步应先在 Reacher 或动作 magnitude 有真实变化的环境重复 oracle gate；只有两侧 specialist 都通过，才恢复多-specialist ensemble。

详细可复现实验记录见 `docs/alrd_validation_2026-07-11.md`。正式训练请优先使用 `--no_checkpoints` 做筛选，并将中间权重导出到 `/dev/shm` 或数据盘；保留的 checkpoint 数量必须受限，避免当前系统盘被 3GB/份 optimizer checkpoint 写满。

## 8. Reacher 迁移状态

已经从 Hub 的真实 metadata 完成 train / ID / OOD action audit。当前数据版本中
train 有 40,000 条 `37 × 2` joint-torque 序列；per-chunk mean torque norm 的 mean
为 `2.0721`、std 为 `0.5756`、CV 为 `0.2778`，median split 为严格平衡的
20,000 / 20,000。action magnitude 因此非退化。ID/OOD action-norm mean 相对 train
的偏移都小于 train std 的 `0.07` 倍，没有明显 marginal norm shift。

认证下载已经补齐并解码 version-consistent feasibility inventory：train 1,000、ID 100、
OOD 100。使用 aggregate metadata 时，low/high visual-motion proxy ratio 只有 `1.0705`，
episode-level torque/motion correlation 只有 `r=0.0546`。因此 magnitude 虽非退化，
却不是有证据的 motion-effect regime axis；low/high specialist partition 在训练前 gate 即失败。
两侧 teacher 均不允许进入 distillation，KD 与 held-out experiment 不得运行。独立的 official
baseline paired-noise diagnosis 仍等待 3.1 GB checkpoint 完整下载，当前不能判断 ceiling effect。

官方 100k-step Reacher checkpoint 随后已完整下载，并在 ID/OOD 上用 8 windows、10 denoising
steps、3 diffusion seeds 完成 paired-noise diagnosis。baseline zero/opposite-torque response
约为 ID `1.37e-3 / 1.13e-3`、OOD `1.42e-3 / 1.15e-3`，seed 方差极小；true-action
MSE 为 ID `3.79e-4`、OOD `1.309e-3`。说明 baseline 已高度 action-responsive，存在
practical ceiling effect。ID scale monotonicity 仅 3/4（`2a` response 反而小于 `1.5a`），
OOD 为 4/4。

本轮最终结论是 **C. specialist hypothesis fails; redesign partition**。这不是 response KD
失败，因为 oracle 前置条件失败后 KD 被正确禁止。本轮唯一推荐下一路线是转 Robot Arm，先从
真实 joint-angle-delta semantics 和 measured motion/failure axis 定义 partition，再重复 diagnosis
与 oracle gate；不在 Reacher 强调 loss 或沿用 median magnitude split。

新增的通用 counterfactual 路径已支持 `0 / -a / 0.25a / 0.5a / 0.75a / 1.5a / 2a`，
并记录 environment-specific semantics、absolute response curve 和 scale monotonicity。
详见 `docs/reacher_action_semantics.md`、`docs/reacher_baseline_diagnosis.md`、
`docs/reacher_oracle_gate.md` 与 `docs/counterfactual_gt_feasibility.md`。
